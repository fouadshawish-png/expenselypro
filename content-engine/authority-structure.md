# Authority Structure

## الهدف
بناء طبقة سلطة موضوعية متماسكة حول المصاريف والميزانية عبر بنية Hub + Cluster.

## 1) Architecture
- Core Hubs:
  - /إدارة-المصاريف/
  - /الميزانية/
  - /تتبع-المصاريف/
  - /الديون/
  - /الادخار/
- Master Guides:
  - /دليل-إدارة-المصاريف-والميزانية/
  - /financial-guide/
  - /money-management-guide/

## 2) Internal Link Rule (إلزامي)
كل مقال يجب أن يحتوي:
1. رابط إلى مقال رئيسي (Pillar أو Hub).
2. رابط إلى /download.html.
3. رابط إلى دليل شامل.
4. رابط إلى مقال فرعي داعم.

الحد الأدنى: 5 روابط داخلية مفيدة.

## 3) Standard Article Template

### الهيكل الثابت
1. Hook نفسي قوي (2–3 أسطر)
2. مشكلة واضحة
3. مثال رقمي عملي
4. إطار عملي (خطوات أو نموذج)
5. مقارنة (متى يلزم)
6. FAQ
7. CTA مزدوج:
   - CTA قراءة: رابط دليل أو مقال محوري
   - CTA تطبيق: رابط تحميل أو Deep Link

### CTA القابلة لإعادة الاستخدام
- قراءة: /دليل-إدارة-المصاريف-والميزانية/
- تحميل: /download.html
- Deep links:
  - /?route=reports
  - /?route=budgets
  - /?route=debt_ledger
  - /?route=goals

## 4) Update Cycle (كل 60 يوم)
1. تحديث أفضل 5 مقالات.
2. إضافة مثال رقمي جديد لكل مقال.
3. إضافة FAQ جديدة أو تحسين FAQ الحالية.
4. تحديث `last_modified_at`.

## 5) Content Upgrade Layer
داخل المقالات الكبرى:
- PDF مجاني
- Checklist تنفيذ
- قالب ميزانية
- نموذج متابعة أسبوعية

## 6) Quality Gate قبل النشر
- عنوان + وصف ميتا واضحان.
- H1 واحد فقط.
- رابط Hub + رابط تحميل + رابط دليل.
- لا تكرار نصي ميكانيكي.
- CTA غير عدواني.
